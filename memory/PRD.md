# Your Analyst — Live Yahoo Finance Fix

## Problem
The deployed site at https://youranalyst.netlify.app/ stopped showing live data.
Root cause: the JS had **no fetch calls at all** — all stock data was hardcoded
constants and the "1-year price chart" was generated by `Math.random()`.
The previous CORS proxy that powered the live fetch (likely `corsproxy.io`) is
now a paid-only service and was silently returning errors → page silently fell
back to hardcoded values without surfacing the failure.

## Fix Implemented (2026-01)
1. **Real Yahoo Finance integration** added to `index.html`:
   - `yahooFetch()` → tries multiple strategies in order
   - `fetchLiveStock()` → parses chart payload (price, dayH/L, 52W H/L, volume, history)
   - `loadStock()` made async — renders cached baseline instantly, then merges
     live data when fetch resolves
   - Price chart now uses real history when available, falls back to random
     generator only if all fetches fail
   - "LIVE" badge now reflects real state: FETCHING / LIVE @ HH:MM / OFFLINE · cached

2. **Netlify serverless function** (`netlify/functions/yahoo.js`):
   - Same-origin `/api/yahoo?symbol=…&range=…&interval=…` endpoint
   - Proxies to `query1.finance.yahoo.com` with a real UA, 12s timeout
   - Adds permissive CORS headers + 60s cache
   - Most reliable strategy (doesn't depend on flaky public proxies)

3. **`netlify.toml`**:
   - Wires `[[redirects]]` so `/api/yahoo` → `/.netlify/functions/yahoo`
   - Declares `publish = "."` and `functions = "netlify/functions"`

4. **Fallback chain** (in `index.html`) — used if the Netlify function is
   unavailable (e.g., local dev):
   - `api.allorigins.win/raw?url=…`
   - `api.allorigins.win/get?url=…` (wrapped contents)
